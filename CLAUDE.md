# Project: Portal Fatigue - KubeCon EU 2026

## Context
This repo supports Abhinav's KubeCon EU 2026 talk on portal fatigue — the developer experience problem of too many internal platforms and dashboards.

## Git Workflow
- Work on `staging` branch. Never push directly to `main`.
- Merge to `main` only after staging tests pass.

## Conventions
- Python as primary language where code is needed.
- YAML for configuration.
- All code files start with a 2-line ABOUTME comment.
